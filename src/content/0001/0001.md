## 현재 Mdit 기술적으로 빡센 것

* [ ] 드래그 앤 드랍 - 구현하려면 tauri drop event랑 dnd-kit이랑 통합해야 되는데 아직 감이 안 옴
* [ ] 폴더 watcher - 폴더 내 파일 변경 있으면 반영 해야 함.
* [ ] AI - 모델 안 좋은 거 쓰면 결과 이상하게 나옴. 프롬프트가 조금 복잡한 것 같다는 생각이 듬.

이 외적으로는 거의 없다. 일단 드래그 앤 드랍부터 구현할건데 그래서 계속 감을 익히고 있다.

## 현재 하고 싶은 것

* [ ] 대충 짐 싸서 배낭 여행 가고 싶다.
* [ ] 라면 먹을까 말까 고민 중이다.

## 이미지

그나저나 지금 이 블로그는 astro가 알아서 이미지를 최적화해줘서 github에 그냥 올리면 되긴 하는데 마크다운 substack의 경우에는 어떻게 처리하면 좋을까?

* 이미지를 그대로 github에 올리면 metadata가 남는다.

  * 이걸 내가 막을 수 있는 방법이 있을까?
  * 빌드할 때 경고 메세지?
* 이미지를 너무 많이 올려서 github 용량 초과하면?

  * 그건 내가 상관할 바가 아니다.
* 최적화는 cloudflare images or @cf-kit/image?

  * @cf-kit/image는 내가 예전에 만든 건데 불완전하다.

나중에 MVP 만들 때 사진은 기능에서 제외하자.

## 감기

감기에 걸려서 기침이 미친 듯이 나왔다. 지금은 본가에 와서 휴양 중인데 그저께는 5시까지 기침하다 겨우 잠들었다. 이번 애는 조금 세다.

내일은 코드블록을 많이 다룬 글을 올려봐야겠다. 아 그리고 지금 느낀건데 frontmatter 개선도 필요하다. 탭이나 방향키로 이동시킬 수 있으면 좋을 것 같다.
